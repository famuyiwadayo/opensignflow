# OpenSignFlow

OpenSignFlow is an open-source, AI-assisted PDF signing and document workflow platform for freelancers, small teams, agencies, and startups.

The project is intentionally designed as both:

1. a serious portfolio project that demonstrates frontend, backend, AI, DevOps, and product skills; and
2. a hosted SaaS product that can be monetized while keeping the core product open source.

## Current status

Planning and architecture documentation phase.

## Product thesis

Small teams often need a simpler and more affordable way to prepare, send, sign, and manage PDF documents. Existing e-signature platforms can be expensive or overly complex. OpenSignFlow will provide a practical open-source alternative with AI-assisted document preparation.

## Initial MVP

The first version should support:

- user registration and authentication;
- document dashboard;
- PDF upload;
- PDF preview;
- drag-and-drop field placement;
- recipients;
- secure signing links;
- signer page;
- signature capture;
- final PDF generation;
- audit trail;
- AI document summary;
- AI signing-field suggestions;
- AI email draft generation.

## Tech stack

| Area | Decision |
|---|---|
| Monorepo | Turborepo + pnpm workspaces |
| Frontend | Next.js, TypeScript, Tailwind CSS, shadcn/ui |
| Frontend state/forms | TanStack Query, TanStack Form, TanStack Table, TanStack Virtual where useful |
| Backend | NestJS, TypeScript |
| Database | PostgreSQL + Prisma |
| Jobs | Redis + BullMQ |
| Storage | S3-compatible storage, local MinIO, production Cloudflare R2/S3 |
| Email | Resend in production, Mailpit locally |
| AI | Provider-abstracted backend AI module |
| API docs | OpenAPI/Swagger generated by NestJS |
